## 22:49 · 轻量模式瞬间停止修复

**问题发现：**
- 恺江发现轻量模式启动很慢，"好像不是关掉，而是让它睡眠"
- 检查代码：`kill_service` 函数用 `proc.terminate()` + `wait(timeout=3)` 优雅停止
- 每个进程等 3 秒，4 个服务需要 12 秒

**屎山代码：**
```python
# 原来的代码（优雅停止，慢）
proc.terminate()  # 先尝试正常停止
proc.wait(timeout=3)  # 等待3秒
# 超时才 proc.kill()
```

**解决方案：**
- 新增 `--instant` 参数，直接 `proc.kill()`，不等待
- 轻量模式使用 `--stop-all --force --instant`
- 瞬间停止，<1 秒完成

**已修改：**
- `lan_service_checker.py`：新增 `instant` 参数
- `lan_lightweight.bat` v2.1：使用 `--instant`

**测试结果：**
```

  - 互联网节点 (PID 39832) 已瞬间停止
```
