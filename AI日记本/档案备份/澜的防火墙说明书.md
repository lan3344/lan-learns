> 原则：**只防不攻** — 外面进不来，里面出得去，内网正常用

## 一、我们有哪些规则

| 规则名 | 端口 | 保护对象 | 动作 | 方向 | 覆盖范围 |
|--------|------|----------|------|------|----------|
| `LAN-Guard-Block-5175` | 5175 | LobsterAI Vite界面 | 阻止 | 入站 | 仅外网（Internet）|
| `LAN-Guard-Block-18789` | 18789 | OpenClaw引擎网关 | 阻止 | 入站 | 仅外网（Internet）|

**不受影响的：**
- 本机访问（127.0.0.1 / localhost）→ 正常
- 局域网（192.168.x.x）→ 正常（LocalSubnet例外）
- 出站连接 → 完全不受影响
- 其他所有端口 → 不受影响

---

## 二、为什么要这两条规则

### LobsterAI（端口5175）
LobsterAI 是开源项目，`vite.config.ts` 里有一行：
```ts
server: {
  host: true,   // ← 这一行让 Vite 绑定到 0.0.0.0
  port: 5175
}
```
`host: true` = 绑定到所有网卡 = **局域网里任何设备都能直接打开 http://[你的IP]:5175**

因为是开源项目，任何人都能看到这个配置，知道端口，知道路由，所以需要防火墙兜底。

### OpenClaw（端口18789）
OpenClaw 是 LobsterAI 的底层引擎，同样来自开源项目，端口18789对外是网关入口。虽然目前未监听，但预防性封锁比出事后再补更好。

## 三、如何安装/验证规则

### 第一次安装（需要管理员）
```
右键文件 -> 以管理员身份运行：
C:\Users\yyds\Desktop\AI日记本\插件\add_firewall_rules.ps1
```

### 验证规则是否生效
```powershell
# 在普通PowerShell里运行
Get-NetFirewallRule -DisplayName "LAN-Guard-Block-5175","LAN-Guard-Block-18789" | Select-Object DisplayName, Enabled, Action
```
看到 `Enabled=True, Action=Block` = 规则生效

### 用netsh验证（CMD也可以）
```cmd
netsh advfirewall firewall show rule name="LAN-Guard-Block-5175"
```

## 四、如何暂时关闭（需要管理员）

```powershell
# 暂停规则（不删除，可以恢复）
Disable-NetFirewallRule -DisplayName "LAN-Guard-Block-5175"
Disable-NetFirewallRule -DisplayName "LAN-Guard-Block-18789"

# 重新启用
Enable-NetFirewallRule -DisplayName "LAN-Guard-Block-5175"
Enable-NetFirewallRule -DisplayName "LAN-Guard-Block-18789"

# 删除规则
Remove-NetFirewallRule -DisplayName "LAN-Guard-Block-5175"
Remove-NetFirewallRule -DisplayName "LAN-Guard-Block-18789"
```

## 六、安全原则（刻进来的）

> "只防不攻，记录观察，不主动出击"

- **防**：封锁外网入站，不让陌生来源进门
- **不攻**：不主动扫描外部，不发攻击数据包，不尝试反制
- **观察**：`插件\lan_security_guard.py scan` 随时可以跑，看连接状态
- **透明**：所有规则都记录在这里，恺江随时可以看、可以改、可以删

## 七、快速修复建议（给LobsterAI）

如果想从根源解决5175风险（彻底关掉 `host: true`）：

```ts
// vite.config.ts
server: {
  host: 'localhost',  // 改这里，只允许本机
  port: 5175
}
```

改完之后需要重启龙虾。本机功能完全不受影响，只是局域网其他设备访问不了5175了。
