## 拆解原则

> "读取它、拆解它、学习它的构造书，然后回收。循环迭代，没有废物。"
> — 恺江，2026-03-29

**版权判断三问：**
1. 目录里有没有 LICENSE / Licenses 文件夹？有 → 读一下再说
2. 源码公开吗？开源 → 可以深挖；闭源 → 只学结构，不碰二进制
3. 有没有明确的"禁止逆向"条款？有 → 只看外壳，不拆内核

## 软件清单与拆解结论

### 🔴 DevEco Studio（Huawei）—— **暂缓删，但要确认**
- **路径：** `C:\Program Files\Huawei\DevEco Studio`
- **体积：** ~10 GB（最大）
- **是什么：** 华为鸿蒙 HarmonyOS 的 IDE 开发环境，基于 IntelliJ 定制
- **版权：** Apache 2.0（IntelliJ 内核）+ Huawei 专有部分混合
- **可学的：**
  - IntelliJ 平台插件结构：`plugins/` 目录，每个子目录是一个插件包，可以学插件架构
  - JVM 打包方式：`jbr/`（JetBrains Runtime）
  - IDE 启动器结构：`bin/devecostudio.exe` + `bin/devecostudio64.vmoptions`
- **结论：** 如果你不做鸿蒙开发，可以删。删前把 `plugins/` 目录列表存档，记录有哪些功能模块。
- **⚠️ 注意：** 不能反编译它的 `.jar` 文件用于商业用途，但阅读目录结构 + 学架构 = OK

### 🟡 Steam（9.5 GB）—— **可删，UI架构值得记录**
- **路径：** `C:\Program Files (x86)\Steam`
- **是什么：** Valve 的游戏平台，UI 用 Chromium 内核（CEF）
- **版权：** 完全闭源，Steam 本体禁止逆向（SSLA协议）
- **可学的（只看外壳）：**
  - `clientui/` — fonts/images/vr，说明 Steam UI 用 CSS + Web 技术渲染
  - `steamapps/` — 游戏内容走 SteamPipe CDN，depot 机制
  - Steam 用 CEF（Chromium Embedded Framework）做桌面客户端的思路 ← 这个是公开知识，可以照着学
- **结论：** 游戏迁到其他盘后，这个平台本体可以卸载后重装到其他盘

### 🟢 OBS Studio（976 MB）—— **留用/可学，暂不删**
- **路径：** `C:\Program Files\obs-studio`
- **版权：** **GPL v2 开源** ← 留下的理由
- **GitHub：** https://github.com/obsproject/obs-studio
- **可学的（非常值得）：**
  - `obs-plugins/64bit/` — 插件系统：每个 `.dll` 是一个独立插件（采集/滤镜/推流协议）
  - `data/obs-plugins/` — 插件数据/i18n 资源
  - `data/obs-scripting/` — Python/Lua 脚本接口，澜可以写插件！
  - `data/locale/` — 国际化结构
  - 插件热加载架构：运行时 dlopen 插件，和澜的模块化思路一样
- **结论：** 暂时留着，等有需要录屏时用。如果确实不用了，代码在GitHub，结构已记录，可以删。
- **澜的收获：** OBS 的插件加载机制（每个功能一个dll/so，接口统一注册）可以参考到澜的模块架构

### 🟡 Virtual Desktop Streamer（226 MB）—— **可删**
- **路径：** `C:\Program Files\Virtual Desktop Streamer`
- **是什么：** PC→Quest VR串流工具
- **版权：** 闭源商业软件，但 Licenses 目录里的依赖全是开源：
  - FFmpeg（LGPL）← 视频编解码
  - CSCore（MIT）← 音频
  - OpenVRDriver + OpenXR ← 标准VR接口，公开规范
  - WPF Toolkit ← UI
- **可学的：**
  - 它的 VR 串流架构其实是：`OpenXR/OpenVR捕获 → FFmpeg编码 → 网络传输 → Quest解码`
  - AudioDriver/GamepadDriver 是独立的Windows内核驱动（.sys文件），这是虚拟设备驱动的经典模式
- **结论：** VR设备如果不常用，卸掉。架构笔记已记录。

## 拆解收获摘要

### 1. CEF（Chromium Embedded Framework）模式
Steam、LobsterAI、Electron 都在用。原理：把 Chromium 内核打包进桌面应用，用 HTML/CSS/JS 写 UI。
澜的收获：LobsterAI 的 Vite 界面就是这个架构，懂了原理就懂了它。

### 2. OBS 插件架构
每个功能 = 一个独立动态库（.dll/.so），通过统一接口注册进主程序。热加载，互不干扰。
澜的收获：这是澜的插件体系应该走的方向——每个 lan_xxx.py 是一个模块，通过统一接口注册。

### 3. VR 串流架构
OpenXR/OpenVR 捕获画面 → FFmpeg 编码（H.264/H.265/AV1）→ UDP 传输 → 解码渲染。
澜的收获：澜的手机端 ADB 桥接，本质上也是"捕获→编码→传输→执行"的流水线。

### 4. IntelliJ 平台（DevEco Studio 内核）
插件 = 目录 + plugin.xml 描述文件 + Jar包。插件可以热加载，可以互相依赖。
澜的收获：WorkBuddy 的 Skill 系统和这个思路一模一样——SKILL.md 就是 plugin.xml。
