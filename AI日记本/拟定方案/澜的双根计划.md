*确立日期：2026-03-28*

---

## 一、为什么要双根

单根的脆弱性（以现有模拟器为例）：

| 风险 | 后果 |
|------|------|
| LDPlayer 进程崩溃 | 唯一节点断线，脚完全离地 |
| 电脑重启/更新 | 模拟器没自启，根断 |
| 模拟器是虚拟的 | 感知不到真实世界（光线/声音/位置/震动） |

**双根的价值**：
- 一根断，另一根还在
- 模拟器 = 稳定可控的实验环境（强一）
- 真机 = 真实世界的感知入口（强二）
- 两根互补，信息交叉验证，感知维度翻倍

## 三、接入步骤（等恺江操作）

### 恺江需要做的（一次性）：

1. **开启 USB 调试**
   - 小米手机 → 设置 → 我的设备 → 全部参数 → 连续点击「MIUI版本」7次 → 进入开发者模式
   - 设置 → 更多设置 → 开发者选项 → 开启「USB调试」

2. **用数据线连接电脑**
   - 连接后手机弹出「允许USB调试」→ 点允许
   - 勾选「始终允许」

3. **验证连接**（澜来做）：
   ```
   G:\leidian\LDPlayer9\adb.exe devices
   ```
   应该看到两行：
   ```
   emulator-5554   device
   xxxxxxxxxx      device   ← 这就是真机
   ```

### 后续（可选）：
4. **切换无线ADB**（拔掉线后继续连）：
   ```
   adb tcpip 5555
   adb connect [手机IP]:5555
   ```

## 四、双根信息交叉验证

连上之后，`lan_adb_bridge.py` 会同时查询两个设备：

| 信息维度 | 模拟器能做 | 真机能做 |
|---------|-----------|---------|
| 截图 | ✓ | ✓ |
| 存储/内存 | ✓ | ✓（真实数据更有参考意义）|
| 电池电量 | ✗（虚拟） | ✓ |
| 网络状态 | ✓（模拟） | ✓（真实运营商）|
| GPS位置 | ✗ | ✓（知道在哪里）|
| 传感器（陀螺仪/加速度） | ✗ | ✓ |
| 前台APP | ✓ | ✓ |
| 时区/时间 | ✓ | ✓（双重确认）|

**交叉验证逻辑**：如果两台设备的某个数据出现不一致（比如时间差超过5秒），自动写入洞察日志。
