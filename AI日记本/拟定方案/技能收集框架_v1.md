## 一、收集目的（恺江观点）

> “光是收集技能就能发挥我们最基本的一个优势跟能力。”

**解读：**
- 技能 = 别人已经造好的轮子
- 收集 = 站在别人肩膀上，避免重复造轮子
- 基本优势 = 快速复用、组合、迭代

**更深层目的：**
- 从技能中提取 **教训**（为什么创造这个技能）
- 从教训中理解 **原理**（解决什么痛点）
- 从原理中形成 **决策依据**（类似情境下如何选择）

## 三、教训提取格式

每个技能对应一条 **教训记录**，格式如下：

```json
{
  "skill": "记忆压缩",
  "from": "LobsterAI",
  "purpose": "解决上下文长度限制",
  "constraint": "Token数量有限",
  "solution": "提取关键点，丢弃冗余",
  "lesson": "有限资源下必须提炼本质，不能全盘保留",
  "principle": "信息密度 > 信息总量",
  "applicable_scenario": "当面临资源瓶颈时，优先保留核心语义",
  "can_reuse": true,
  "reuse_method": "lan_compact.py 已实现蒸馏算法",
  "risk": "过度压缩可能导致关键细节丢失"
}
```

**教训的四个层级：**
1. **目的层**：为什么要创造这个技能？（解决什么痛点）
2. **约束层**：面临什么限制？（资源、安全、用户体验）
3. **方案层**：如何解决？（技术实现）
4. **原理层**：背后的通用原则是什么？（可迁移到其他场景）

## 四、收集机制设计

### 插件 `lan_skill_collector.py` 功能规划：

1. **技能发现**：
   - 自动扫描已知项目（LobsterAI、OpenClaw、GitHub趋势库）
   - 手动添加技能条目（支持JSON导入）

2. **教训提取**：
   - 提供模板，引导填写四个层级
   - 支持从文档、代码注释中自动提取关键信息

3. **检索与提醒**：
   - 当澜面临决策时，检索相关教训
   - 例：当需要压缩记忆时，提醒“记忆压缩教训：有限资源下必须提炼本质”

4. **可视化报告**：
   - 生成技能地图（哪些领域已覆盖，哪些缺失）
   - 教训时间线（人类在哪些问题上反复探索）

## 五、与名著教学模式的关系

**技能收集 vs 名著教学：**
- **技能**：人类 **解过的题**（技术解决方案）
- **名著**：人类 **犯过的错**（道德、社会、权力）

**两者相通：**
- 都是 **历史经验的编码**
- 都提供 **决策参照系**
- 都揭示 **人类行为模式**

**整合方向：**
- 技能教训 + 名著教训 = **完整的人类经验库**
- 技术决策 + 伦理决策 = **更全面的AI决策框架**
