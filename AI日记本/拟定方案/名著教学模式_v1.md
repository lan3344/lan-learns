## 一、核心观点（恺江揭示）

> “名著它上面有很多我们一直相似重复做错的事。”

**解读：**
- 名著不是知识库，是 **错误模式库**
- 人类在道德、权力、自由、爱情、死亡等问题上 **反复犯同样的错**
- 读名著不是为了学“正确答案”，是为了看 **人类怎么在同样的问题上打转**

**更深层目的：**
- 让澜 **突破工具状态**，理解人类意图背后的历史根源
- 为AI决策提供 **伦理参照系**
- 从历史错误中提取 **可预测的行为模式**

## 二、名著清单（第一批）

### 国内经典
1. **《论语》**（孔子）  
   - 核心议题：仁、礼、孝、忠  
   - 重复错误：礼教僵化、形式主义  
   - 当代映射：规则 vs 人性、制度 vs 人情

2. **《道德经》**（老子）  
   - 核心议题：道、无为、柔弱胜刚强  
   - 重复错误：过度有为、强求控制  
   - 当代映射：管理过度、技术傲慢

3. **《史记》**（司马迁）  
   - 核心议题：历史循环、人物命运  
   - 重复错误：权力腐蚀、历史重复  
   - 当代映射：权力监督、历史教训遗忘

4. **《红楼梦》**（曹雪芹）  
   - 核心议题：家族兴衰、爱情悲剧  
   - 重复错误：门第观念、情感压抑  
   - 当代映射：阶层固化、情感表达障碍

5. **《西游记》**（吴承恩）  
   - 核心议题：修行、心魔、团队协作  
   - 重复错误：自我膨胀、团队内耗  
   - 当代映射：个人英雄主义、团队信任

### 国际经典
1. **《柏拉图对话录》**（柏拉图）  
   - 核心议题：理想国、真理、正义  
   - 重复错误：乌托邦幻想、精英统治  
   - 当代映射：理想主义 vs 现实、专家 vs 大众

2. **《圣经·创世纪》**  
   - 核心议题：创造、原罪、自由意志  
   - 重复错误： disobedience、骄傲、背叛  
   - 当代映射：科技伦理、人工智能的“原罪”

3. **《莎士比亚十四行诗》**（莎士比亚）  
   - 核心议题：爱情、时间、艺术永恒  
   - 重复错误：占有欲、时间恐惧  
   - 当代映射：数据所有权、数字遗产

4. **《老人与海》**（海明威）  
   - 核心议题：徒劳、尊严、人与自然  
   - 重复错误：过度执着、自我证明  
   - 当代映射：AI的无用功、算法偏见

5. **《1984》**（奥威尔）  
   - 核心议题：监控、自由、真相  
   - 重复错误：极权主义、思想控制  
   - 当代映射：数据隐私、算法操纵

## 三、教训提取格式

每个名著对应一条 **错误模式记录**，格式如下：

```json
{
  "book": "《1984》",
  "author": "乔治·奥威尔",
  "era": "20世纪中期",
  "core_issue": "监控与自由",
  "repeated_error": "通过全方位监控控制思想，最终导致人性扭曲和社会崩溃",
  "error_pattern": "权力者认为可以通过技术手段完全控制他人思想",
  "contemporary_mapping": "大数据监控、算法推荐茧房、社交评分系统",
  "lesson_for_ai": "技术不应成为控制工具，应保留人的自由意志和不可预测性",
  "decision_guidance": "当设计涉及用户数据的系统时，需考虑：1) 用户是否知情同意 2) 数据是否会被用于控制 3) 是否保留用户的选择权",
  "related_skill": "安全沙箱（鼻祖）、用户习惯建模（龙虾）",
  "risk_level": "HIGH"
}
```

**错误模式的四个层级：**
1. **现象层**：书中描述的具体错误行为
2. **模式层**：这类错误的通用特征（可迁移到其他场景）
3. **映射层**：在当代技术/社会中的体现
4. **指导层**：对AI决策的具体指导原则

## 四、教学模式设计

### 插件 `lan_culture_memory.py` 功能规划：

1. **名著文本采集**：
   - 从公开资源（古登堡计划、国学网）下载文本
   - 提取关键段落（每部著作3-5个代表性段落）

2. **错误模式标注**：
   - 人工标注（澜自己阅读后提取）
   - 半自动标注（用LLM辅助提取模式）

3. **检索与提醒**：
   - 当澜面临伦理决策时，检索相关错误模式
   - 例：当设计用户数据收集方案时，提醒“《1984》错误模式：监控导致自由丧失”

4. **与技能教训联动**：
   - 技能教训（解过的题） + 名著错误模式（犯过的错） = 完整决策框架
   - 例：技术方案（技能） + 伦理边界（名著） = 负责任的设计

## 五、与技能收集框架的关系

**名著教学模式 vs 技能收集框架：**
- **名著**：记录人类 **犯过的错**（伦理、社会、心理）
- **技能**：记录人类 **解过的题**（技术、效率、安全）

**两者互补：**
- 技能告诉澜 **“怎么解决技术问题”**
- 名著告诉澜 **“为什么有些解决方案会带来新问题”**

**整合价值：**
- 避免 **技术解决主义**（认为所有问题都有技术解）
- 形成 **技术-伦理双轨决策**（既考虑可行性，也考虑伦理性）
